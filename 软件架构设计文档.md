## 1. 概述

### 1.1 项目背景

AI-Psychological-Assistant-System（AI心理助手系统）是一个面向心理健康领域的智能辅助平台，旨在通过人工智能技术为用户提供心理支持和咨询服务。

### 1.2 项目目标

- 提供专业的心理健康知识文章管理
- 支持用户与AI心理咨询师的对话交互
- 记录和分析用户的情绪状态变化
- 提供数据可视化的分析仪表盘

### 1.3 业务范围

| 模块     | 功能描述             |
| -------- | -------------------- |
| 用户认证 | 登录、注册、权限管理 |
| 数据分析 | 仪表盘统计展示       |
| 知识管理 | 文章分类、发布、管理 |
| 咨询服务 | AI对话、咨询记录     |
| 情绪追踪 | 情绪日志记录与分析   |

## 2. 架构表示

### 2.1 架构风格

采用**集成式单体应用 (Integrated Monolith)** 架构风格，结合**MVC模式**和**组件化开发**思想。

**架构层次：**

| 层次   | 职责               | 技术实现             |
| ------ | ------------------ | -------------------- |
| 表示层 | UI展示、用户交互   | Vue 3 + Element Plus |
| 业务层 | 状态管理、路由控制 | Pinia + Vue Router   |
| 数据层 | API调用、数据请求  | Axios                |

### 2.2 架构模式

- **MVVM模式**：Vue 3采用的架构模式，实现数据与视图的双向绑定
- **服务层模式**：通过API模块统一管理后端接口调用
- **状态管理模式**：使用Pinia进行全局状态管理

## 3. 架构因素

### 3.1 质量属性

| 属性         | 描述                   | 实现策略                     |
| ------------ | ---------------------- | ---------------------------- |
| **可用性**   | 系统应7x24小时可用     | 无状态设计、错误重试机制     |
| **性能**     | 页面加载时间<2秒       | 懒加载、代码分割、缓存策略   |
| **可维护性** | 代码易于理解和修改     | 模块化设计、清晰的代码结构   |
| **可扩展性** | 支持功能扩展和集成     | 插件化架构、API标准化        |
| **安全性**   | 保护用户数据和系统安全 | Token认证、权限控制、XSS防护 |

### 3.2 约束条件

| 约束类型 | 描述                               |
| -------- | ---------------------------------- |
| 技术约束 | 必须使用Vue 3 + Element Plus技术栈 |
| 时间约束 | 项目周期6个月                      |
| 资源约束 | 前端开发人员2-3人                  |
| 合规约束 | 符合数据隐私保护法规               |

### 3.3 设计原则

1. **单一职责原则**：每个组件/模块只负责一个功能
2. **开闭原则**：对扩展开放，对修改关闭
3. **依赖倒置原则**：依赖抽象而非具体实现
4. **接口隔离原则**：使用细粒度接口
5. **迪米特法则**：组件间保持最小耦合

## 7. 进程视图

### 7.1 进程结构

系统采用**单页应用(SPA)**架构，所有功能在同一浏览器进程中运行。

**主要进程/线程：**

| 进程/线程     | 职责             | 说明                     |
| ------------- | ---------------- | ------------------------ |
| 主线程        | UI渲染、用户交互 | Vue组件渲染、DOM操作     |
| 事件循环      | 处理用户事件     | 点击、输入等交互事件     |
| HTTP请求线程  | 异步数据请求     | Axios发起的网络请求      |
| WebSocket连接 | 实时通信         | 可选，用于AI对话实时推送 |

### 7.2 进程交互

```
┌───────────────┐      ┌───────────────┐      ┌───────────────┐
│   浏览器主线程  │      │   事件循环     │      │   HTTP请求     │
│   (UI渲染)     │◀─────│   (事件处理)   │─────▶│   (数据获取)   │
└───────────────┘      └───────────────┘      └───────────────┘
         │                                            │
         │ 渲染结果                                    │ 响应数据
         ▼                                            ▼
┌───────────────┐                              ┌───────────────┐
│   用户界面     │◀──────────────────────────────│   后端服务    │
└───────────────┘                              └───────────────┘
```

### 7.3 典型场景流程

**场景：用户登录**

```
1. 用户打开登录页面
    │
    ▼
2. Vue Router渲染login.vue组件
    │
    ▼
3. 用户输入用户名和密码
    │
    ▼
4. 点击登录按钮触发handleLogin方法
    │
    ▼
5. 调用API.login()发送POST请求
    │
    ▼
6. axios拦截器添加Authorization头
    │
    ▼
7. 后端验证用户信息
    │
    ▼
8. 返回Token
    │
    ▼
9. localStorage存储Token
    │
    ▼
10. 路由跳转到首页
```

**场景：AI咨询对话**

```
1. 用户进入咨询页面
    │
    ▼
2. 加载历史咨询记录
    │
    ▼
3. 用户输入咨询内容
    │
    ▼
4. 发送POST请求到AI服务
    │
    ▼
5. AI服务生成回复
    │
    ▼
6. 返回回复内容
    │
    ▼
7. 更新聊天界面
    │
    ▼
8. 保存咨询记录到数据库
```

### 7.4 并发与同步

| 场景         | 并发处理         | 同步机制      |
| ------------ | ---------------- | ------------- |

| 路由切换     | 等待当前请求完成 | 导航守卫      |
